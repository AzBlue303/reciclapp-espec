# ADR-2: Tecnología para desarrollo móvil.

## Situación
Propuesto.

## Contexto
Al realizar una aplicación móvil para uso público no hay control sobre qué tipo de dispositivo ocuparan los usuarios (Android o IOs), por lo que el sistema debe ser multiplataforma.

## Decisión
Hemos decidido utilizar Ionic con Angular, dado que permite crear interfaces de usuario flexibles para todas las principales plataformas de aplicaciones desde una única base de código compartida.

## Consecuencias
Utilizar Ionic puede generar problemas de rendimiento, lo que puede afectar la experiencia del usuario. Sin embargo evita la necesidad de realizar las aplicaciones de manera nativa.

## Cumplimiento
Se implementarán funcionalidades que el mismo Ionic provee, las cuales mejoran la carga de la aplicación.
